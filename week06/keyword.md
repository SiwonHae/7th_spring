### 지연로딩과 즉시로딩의 차이
- 지연 로딩은 실제로 데이터가 필요할 때 조회하고, 즉시 로딩은 엔티티가 로드될 때 연관된 데이터도 즉시 함께 조회한다.
### Fetch Join
- 연관딘 엔티티를 한번의 쿼리로 함께 조회하는 방법으로, JOIN과 FETCH 키워드를 사용해서 지연 로딩 관계를 즉시 로딩처럼 작동하게 하고 N + 1 문제를 해결할 수 있다.
### @EntityGraph
- JPA에서 특정 엔티티를 조회할 때 연관된 엔티티를 미리 로드하는 방법으로, 어노테이션으로 필드를 지정하여 Fetch Join과 유사하게 작동하도록 한다.
### JPQL
- JPA에서 엔티티 객체를 대상으로 하는 객체 지향 쿼리 언어이다. SQL과 유사한 문법을 사용하지만, 데이터베이스 테이블 대신 엔티티의 속성을 참조한다.
### QueryDSL
- 도메인 모델을 기반으로 쿼리 객체를 생성하여 직관적으로 쿼리를 작성할 수 있도록 도와주는 라이브러리이다.
### N+1 문제를 해결할 수 있는 여러 가지 다른 방법
- Fetch Join 사용, @EntityGraph 사용, Batch Fetching 사용, @BatchSize 사용, DTO 프로젝션 사용, QueryDSL 사용
